Mac 的 Force Touch

发表于 2016-09-24   |     |   阅读次数

用了一年多，但仍然无法相信 Force Touch 如此真实的按下体验居然是模拟出来的。

Force Touch 的原理

Force Touch触控板并不会移动，但你用手指按下它时依然能够听到和感觉到点击，这得益于其所具备的触觉反馈能力。当你点击触控板时，其表面并不会下沉，而是产生一次震动，就像是PlayStation或Xbox游戏手柄的力反馈一样。

实际使用中，触控板并不是纹丝不动的。触控板因为受力，会稍微往下形变，但是形变量很小。触控板的震动反馈有3档可以调节，还附有一个静默模式，能够降低点击反馈的声音（除了振动的声音，按键点击的清脆反馈音应该是喇叭播放的）。触控板按下会分为轻按和重按模式，轻按就类似传统触控板的一次点击，不过更为平滑、干脆；只需再施加少许力气，触控板会再一次震动，给你第二段按下的反馈。

硬件和软件的完美结合

如果说仅仅只是一个好用的触控板，也不至于让人震撼。震撼的是，使用过程中，我又发现了许多魔法般的交互，自然而且直觉。

  • 对一个文件／文件夹的名字部分重按，能够快速进入重命名状态
  • 对一个文件的缩略图部分重按，能弹出快速预览窗口
  • 弹出的预览窗口，轻按不会有影响，重按可以关闭预览窗口
  • 除了轻按（Click）、重按（Force Click）之外，传统的轻触（Tap）也依然有用，比如浏览器双指两次轻触放大
  • Quickplay 的快进支持多级按下，连着 嗑嗑嗑 的声音太爽了
  • 单指重按单词快速查词典、Safari重按网页快速预览（虽然加载很慢）

太棒了。。。

交互设计如何提升用户的操纵感

发表于 2016-08-26   |     |   阅读次数

平时在讨论汽车的体验时，总会提到汽车的操控感，价位高的汽车往往能提供给用户更加强烈的操纵感。汽车的操纵感来源于响应用户的操作，当汽车对用户每一次操作，比如转向、刹车、加速等，都能精准地响应，符合用户直觉之中的设想时，用户便会觉得这辆车操纵感很棒。在交互设计中，同样也要给用户提供精准的响应，来满足用户的操纵感。而且，用户的操作来得比汽车操作更为频繁，操纵感对于用户体验的影响也更为显著。应用应该为用户的每一次输入，比如点击、滑动、手势等，提供符合用户所设想的反馈，满足用户的直觉，提升用户的操纵感。要讨论如何给用户更高的操纵感，我们可以从每个不同的操作说起，来探讨用户在操作的过程中所想的是什么，预期的反馈是怎么样的。

React的进化？深入理解React Fiber（一）

发表于 2016-08-12   |     |   阅读次数

最近知乎上大家在讨论 React Fiber（如何理解React Fiber架构），刚好最近有在看React的实现方式的文章，这里可以讨论一下 React Fiber 究竟比 React 进化了什么，以及它的实现思路和未来，本文大量参考了 React 团队的文档、Github Issues、PR时候的讨论等。

Fiber 能解决 React 渲染大块内容时候的卡顿

React Fiber 是 React 团队正在着手重写的 React 核心算法（Core algorithm），重写后的核心 Feature 是增量渲染（Incremental rendering），能够优先渲染重要内容，次要部分随后再增量更新。Fiber 通过把需要很长时间渲染的内容分解成小块，分散到每一帧的间隔去处理，提升 React 对需要快速反映的动画、用户操作的响应速度。
